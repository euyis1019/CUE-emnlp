LLMs exhibit puzzling heterogeneity when reasoning about code: the same model may effortlessly trace variable assignments yet falter on arithmetic composition, or confidently select the wrong conditional branch while failing to converge on boolean short-circuits. Task-level accuracy alone cannot explain these differences, because models with similar performance may fail in internally distinct ways. We argue that the critical question is not just \textit{what} a model encodes at each layer, but whether the answer has been organized into a form the model itself can stably decode. To operationalize this distinction, we introduce a dual diagnostic framework pairing layer-wise linear probing (\textit{information availability}) with Context-Stripped Decoding (\textit{information readiness}), applied to six synthetic code task families spanning data-flow and control-flow primitives. Across five model families and scales from 0.5B to 14B parameters, we uncover a structured internal lifecycle: models first undergo a \textbf{brewing} process in which the answer becomes externally readable before it is self-decodable, followed by divergence into four \textbf{resolution outcomes}---Resolved, Overprocessed, Misresolved, and Unresolved---each validated through targeted causal interventions. Crucially, different code primitives induce distinct outcome fingerprints: data-flow tasks fail primarily through \textit{overprocessing} (correct answers formed then destroyed), while control-flow tasks fail through \textit{misresolution} or \textit{non-convergence}. Computationally equivalent Loop and Loop-unrolled tasks produce sharply separated fingerprints, demonstrating that models are sensitive to code syntax, not just underlying computation. The brewing scaffold remains stable across architectures and scales; what changes is only the resolution success rate---mechanism and capability are separable.
\end{abstract}
