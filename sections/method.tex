\paragraph{Benchmark.}
To study internal code reasoning, we need tasks that are simple enough to isolate specific reasoning primitives yet rich enough to produce distinct computational behaviors. We therefore design six task families along a classical axis from program analysis: \textbf{data flow} versus \textbf{control flow} \citep{aho2006compilers, allen1970control, kildall1973unified, nielson1999principles}. Each instance consists of a short code snippet $C$ paired with a question suffix $Q$, forming a source prompt $S = [C;\,Q]$. The answer is always a single digit $t^* \in \mathcal{D} = \{0,1,\dots,9\}$, ensuring a one-token target and removing confounds from multi-token decoding. The six families are: (1)~Copying, (2)~Computing, (3)~Conditional, (4)~Loop, (5)~Loop-unrolled, and (6)~Short-circuit. Code snippets are kept intentionally short so that layer-wise signals reflect reasoning primitives rather than long-context effects; restricting the answer to a single digit token (0--9) provides a shared output space for the diagnostic functions defined below, enabling direct cross-layer comparison. Full details and example cases are given in \cref{app:benchmark}.

\subsection{Dual Diagnostic Framework}
\label{subsec:dual_diagnostic}

Given a decoder-only transformer $\mathcal{M}$ with $L$ layers, let $\mathbf{h}^\ell \in \mathbb{R}^d$ denote the last-token hidden state at layer $\ell$ when the model processes the source prompt $S$. Because the final position attends to the full preceding context, it serves as a compact representation of what the model has computed up to that depth. We define two diagnostic functions $\Phi_{\mathrm{P}}^\ell, \Phi_{\mathrm{C}}^\ell: \mathbb{R}^d \to \Delta^{|\mathcal{T}|}$ that map $\mathbf{h}^\ell$ onto the probability simplex over the classification space $\mathcal{T} = \mathcal{D} \sqcup \{\bar{d}\}$, where $\bar{d}$ aggregates all non-digit tokens into a single residual class ($|\mathcal{T}| = 11$). We write $\sigma(\cdot) \triangleq \mathrm{softmax}(\cdot)$ throughout.

% 中文: 第一个诊断是 linear probing。对每一层训练一个逻辑分类器 Φ_P^ℓ(h^ℓ) = σ(W^ℓ h^ℓ + b^ℓ)，测试答案信息是否已经在 hidden state 中线性可读。这问的是一个必要条件：答案是否已经以简单形式存在？为避免 probe 被表面分布线索投机取巧，我们在十个 digit 类之外引入一个 near-OOD residual 类及对应训练样本（详见附录）。

\paragraph{Linear Probing.}
For each layer we train a logistic classifier
%
\begin{equation}
\label{eq:probing}
\Phi_{\mathrm{P}}^\ell(\mathbf{h}^\ell) = \sigma(W^\ell \mathbf{h}^\ell + \mathbf{b}^\ell),
\end{equation}
%
which tests whether answer information is already linearly recoverable from the hidden state. This diagnostic asks a necessary-condition question: \emph{is the answer already present in a simple form?} To prevent the probe from exploiting superficial distributional cues, we augment the ten digit classes with a near-OOD residual class and corresponding training examples (details in \cref{app:probing}).

% 中文: 但信息存在不等于信息可用。为此我们构建第二个诊断：Context-Stripped Decoding (CSD)，基于 Patchscopes。具体做法是从 source run 中提取 h^ℓ，注入到一个只包含问题后缀的 target prompt T = Q 中，移除原始代码上下文 C。模型随后从第 ℓ+1 层到第 L−1 层继续前向传播，注意力上下文仅限于 T。为消除 target prompt 自身的 language prior confound，我们对 patching 后的 logit 做 baseline subtraction：
% Φ_C^ℓ(h^ℓ) = σ(W_u ∘ LN ∘ F_{L-1} ∘ ⋯ ∘ F_{ℓ+1}(h^ℓ|_T) − z_b)|_T
% 其中 z_b 是对 T 不做任何 patching 直接前向传播得到的 logit，代表 language prior。若 argmax Φ_C^ℓ(h^ℓ) = t*，则 h^ℓ 是 self-contained 的：模型自身的解码管道无需回看代码即可产出正确答案。在 Patchscopes 框架中对应配置 (Q, n_Q, I, M, ℓ)。

\paragraph{Context-Stripped Decoding (CSD).}
Information being present does not imply information being usable. We therefore build a second diagnostic on top of Patchscopes \citep{ghandeharioun2024patchscopes}. We extract $\mathbf{h}^\ell$ from the source run and inject it into a target prompt $T = Q$ that contains only the question suffix, thereby removing the original code context $C$. The model then continues its forward pass from layer $\ell{+}1$ through layer $L{-}1$, with attention context restricted to $T$. To eliminate the confound of the target prompt's own language prior, we apply baseline logit subtraction:
%
\begin{equation}
\label{eq:csd}
\Phi_{\mathrm{C}}^\ell(\mathbf{h}^\ell) = \sigma\!\Big(W_u \circ \mathrm{LN} \circ F_{L-1} \circ \cdots \circ F_{\ell+1}\!\big(\mathbf{h}^\ell\big|_T\big) \;-\; \mathbf{z}_b\Big)\bigg|_{\mathcal{T}}
\end{equation}
%
where $\mathbf{z}_b$ is the logit obtained by running $T$ without any patching, representing the language prior. If $\arg\max \Phi_{\mathrm{C}}^\ell(\mathbf{h}^\ell) = t^*$, then $\mathbf{h}^\ell$ is \emph{self-contained}: the model's own decoding pipeline can produce the correct answer without attending back to the code. Within the Patchscopes framework this corresponds to the configuration $(Q,\, n_Q,\, \mathbb{I},\, \mathcal{M},\, \ell)$.

% 中文: Φ_P^ℓ 衡量 information availability，Φ_C^ℓ 衡量 information readiness；沿深度扫描两者的一致与分歧，即可追踪代码推理在模型内部的展开过程。

\paragraph{The key asymmetry.}
$\Phi_{\mathrm{P}}^\ell$ measures \textbf{information availability}---whether an external linear readout can extract the answer; $\Phi_{\mathrm{C}}^\ell$ measures \textbf{information readiness}---whether the model itself can decode the answer once the original context is stripped away. Scanning their agreement and disagreement across depth allows us to trace how code reasoning unfolds inside the model.
